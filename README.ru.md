# Business Relevance Analyzer

*🌐 [English version](README.md)*

AI-скилл для **Claude**, который превращает любой бизнес-источник — статью, лендинг конкурента, стратегический меморандум, транскрипт звонка, письмо от партнёра — в приоритизированный разбор **конкретно для вашего бизнеса**.

Это не саммаризатор. Это такой senior-аналитик, который:

- отделяет сигнал от шума,
- ранжирует находки по бизнес-значимости,
- раскладывает их по функциям (продукт, продажи, операции, стратегия и т.д.),
- предлагает конкретные next steps,
- и честно говорит, где доказательства слабые или идея не переносится на ваш рынок.

Сделан для руководителей, которым нужен **разбор на полстраницы**, а не стена текста.

---

## Как это выглядит на практике

**Вы:** *кидаете статью про раунд конкурента*

**Claude:** выдаёт структурированный разбор — использованный режим, сила сигнала, топ-3 инсайта (у каждого скор, однострочное "почему важно", конкретный next step и caveat), action-лист Act/Validate/Monitor/Ignore и главные риски.

Всё это — привязано к **вашему бизнес-контексту**. Если вы управляете real-estate lead-gen платформой, скилл не предложит вам "изучить возможности в B2B SaaS" — он знает, что это не про вас.

---

## Для кого

Руководители бизнес-юнитов, product и strategy leads, chief of staff, bizdev — все, кто регулярно получает документы, статьи и транскрипты и должен понимать: *"что это значит для нас и что с этим делать?"*

Скилл по умолчанию настроен на контекст real-estate классифайда / lead-gen платформы (см. `references/business-context.md`). **Этот файл нужно отредактировать под свой бизнес** после установки — это обычный markdown, писать код не нужно.

## Установка

Скилл работает как с **Claude Code** (терминальная версия), так и с **Claude.ai** (веб). Выбирайте то, чем пользуетесь.

### Вариант A — Claude Code (терминал)

1. **Если Claude Code ещё не установлен**, поставьте: [https://claude.com/claude-code](https://claude.com/claude-code)

2. **Склонируйте или скачайте репозиторий:**
   ```
   git clone https://github.com/d-alexeev/business-relevance-analyzer.git
   ```
   (Или нажмите зелёную кнопку "Code" на GitHub → "Download ZIP" и распакуйте.)

3. **Поместите скилл туда, где Claude Code ищет скиллы:**
   ```
   ~/.claude/skills/business-relevance-analyzer/
   ```
   Т.е. папку `business-relevance-analyzer`, которую только что склонировали, скопируйте в `~/.claude/skills/`. Если папки `skills` нет — создайте.

4. **Отредактируйте бизнес-контекст** под себя. Откройте:
   ```
   ~/.claude/skills/business-relevance-analyzer/references/business-context.md
   ```
   Замените контекст про недвижимость на описание своего бизнеса, приоритеты и функции, с которыми работаете. Этот файл — главный драйвер релевантности разборов. **Не пропускайте этот шаг.**

5. **Запустите новую сессию Claude Code и попробуйте.** Просто вставьте статью или меморандум и напишите что-то вроде *"Разбери это с точки зрения моего бизнеса"* или *"What matters in this for us, and what should I do?"*. Claude подхватит скилл автоматически.

### Вариант B — Claude.ai (веб)

1. Зайдите на [Claude.ai](https://claude.ai) и авторизуйтесь.

2. Создайте новый **Project** (левая панель → Projects → New project).

3. Загрузите в проект файлы скилла:
   - `SKILL.md`
   - Всё содержимое папки `references/` (`business-context.md`, `modes.md`, `examples.md`)

4. В поле **Custom instructions** проекта вставьте:
   > You have access to a skill called Business Relevance Analyzer. Whenever I share a document, article, transcript, memo, or any other source and ask you to analyze it, follow the instructions in SKILL.md. Always load references/business-context.md first.

5. **Отредактируйте `business-context.md`** (в файлах проекта) под свой бизнес — то же, что в шаге 4 для Claude Code.

6. Готово. Кидайте источники в чат и просите разбор.

## Как пользоваться

После установки — просто отправляйте Claude источник и спрашивайте, что с ним делать. Несколько примеров:

- *"Проанализируй эту статью про конкурента."*
- *"Review this internal memo — I'm taking it to leadership next week, tell me where it's weak."*
- *"Вот транскрипт звонка с клиентом, вытащи сигналы для продукта и операций."*
- *"Can we actually apply this in our business, or is this US-only? Stress-test it."*

Скилл поддерживает несколько **режимов** и выбирает подходящий автоматически. Можно задать явно:

- **Competitive / Market Signal** — для новостей, страниц конкурентов, индустриальных движений
- **Internal Memo Review** — для review своих или чужих стратегических документов
- **Customer Insight** — для транскриптов звонков, интервью с клиентами
- **Applicability Stress Test** — чтобы stress-test, переносится ли идея на ваш бизнес

По умолчанию выдача **компактная** (~полстраницы). Если нужен развёрнутый разбор — с функциональным маппингом, гипотезами и анализом переносимости — попросите "**полный разбор**" или "**deep analysis**".

## Что в репозитории

- `SKILL.md` — сам файл скилла (то, что читает Claude, чтобы понять, как себя вести)
- `references/business-context.md` — бизнес-контекст (редактируйте!)
- `references/modes.md` — подробное описание режимов
- `references/examples.md` — примеры хороших и плохих разборов
- `evals/` — тестовые кейсы и синтетические материалы, использованные при разработке (опционально, для справки)

## FAQ

**Я не технарь. Нужно ли что-то кодить?** Нет. Вы редактируете один файл markdown (`business-context.md`), всё остальное делает сам скилл.

**Работает ли на русском / английском / других языках?** Да. Скилл отвечает на том же языке, на котором к нему обратились. Протестировано на русском и английском.

**Уходят ли мои данные куда-то?** Сам скилл — это просто набор markdown-инструкций. Всё, что обычно Claude делает с вашими данными (по условиям Anthropic), происходит и здесь.

**Можно ли форкнуть и адаптировать под свою команду?** Да, это и есть предполагаемый use case. Форкайте, правьте `business-context.md` под свою компанию, делитесь с командой.
